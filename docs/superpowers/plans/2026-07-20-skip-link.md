# Lien d'évitement (skip link) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ajouter un lien d'évitement ("skip link") accessible au clavier qui permet de sauter directement au contenu principal de chaque page.

**Architecture:** Un seul point d'entrée (`src/layouts/Layout.astro`) rend déjà toutes les pages. Le changement se limite à ce fichier : ajouter le lien comme premier enfant de `<body>`, transformer le conteneur de contenu en `<main id="main-content">`, et styliser le lien pour qu'il soit invisible sauf au focus clavier.

**Tech Stack:** Astro (fichiers `.astro`), Tailwind CSS (classes utilitaires `sr-only` / `focus:*`), pas de framework de test JS configuré dans ce projet (`package.json` n'a que `dev`/`build`/`preview`/`astro`) — la vérification se fait via `npm run build` (compilation) + vérification manuelle au clavier dans le navigateur.

## Global Constraints

- Le lien doit être invisible en usage normal et ne doit avoir aucun impact visuel pour les utilisateurs qui n'utilisent pas le clavier pour naviguer (spec: "Objectif").
- Le lien doit devenir visible et positionné en haut à gauche dès qu'il reçoit le focus clavier (spec: "Objectif").
- Le texte du lien est exactement : `Aller au contenu principal` (spec: "Conception retenue", point 2).
- La cible est `<main id="main-content">`, qui remplace le `<div class="flex-1" style="contain: layout;">` existant à `src/layouts/Layout.astro:84`, en conservant `class="flex-1" style="contain: layout;"` (spec: "Conception retenue", point 1).
- Le style du lien réutilise les tokens de couleur existants du thème : `bg-secondary text-primary-contrast` (spec: "Conception retenue", point 3).
- Un seul lien d'évitement — pas de lien secondaire vers le menu de navigation (spec: "Hors périmètre").

### Task 1: Ajouter le lien d'évitement et la balise `<main>`

**Files:**
- Modify: `src/layouts/Layout.astro:82-88`

**Interfaces:**
- Consumes: rien (aucune dépendance sur d'autres tâches — c'est la seule tâche du plan).
- Produces: un ancrage `#main-content` sur l'élément `<main>`, ciblé par le lien `<a href="#main-content">`.

- [ ] **Step 1: Lire l'état actuel du fichier**

Le bloc `<body>` actuel (`src/layouts/Layout.astro:82-88`) :

```astro
  <body class="flex min-h-screen flex-col">
    {isHomePage ? <HomeHeader /> : <Header />}
    <div class="flex-1" style="contain: layout;">
      <slot />
    </div>
    <Footer />
  </body>
```

- [ ] **Step 2: Remplacer ce bloc**

Remplacer exactement ce bloc par :

```astro
  <body class="flex min-h-screen flex-col">
    <a
      href="#main-content"
      class="sr-only focus:not-sr-only focus:fixed focus:top-2 focus:left-2 focus:z-50 focus:rounded focus:bg-secondary focus:px-4 focus:py-2 focus:font-bold focus:text-primary-contrast focus:outline focus:outline-2 focus:outline-offset-2 focus:outline-primary-contrast"
    >
      Aller au contenu principal
    </a>
    {isHomePage ? <HomeHeader /> : <Header />}
    <main id="main-content" class="flex-1" style="contain: layout;">
      <slot />
    </main>
    <Footer />
  </body>
```

- [ ] **Step 3: Vérifier que le build passe**

Run: `npm run build`
Expected: build réussi, sans erreur (Astro valide le HTML/JSX à la compilation ; une balise `<main>` mal fermée ou un attribut invalide ferait échouer ce build).

- [ ] **Step 4: Vérification manuelle au clavier**

Run: `npm run dev`, puis ouvrir `http://localhost:4321/` dans un navigateur.

1. Cliquer une fois dans la barre d'adresse du navigateur pour donner le focus à la page, puis appuyer sur `Tab`.
2. Expected: un bouton/lien "Aller au contenu principal" apparaît en haut à gauche de la page, avec un fond jaune (`bg-secondary`) et un texte sombre (`text-primary-contrast`) bien contrasté.
3. Appuyer sur `Entrée`.
4. Expected: le focus clavier se déplace sur le `<main>` (le lien disparaît, l'URL affiche `#main-content`) ; répéter `Tab` continue la navigation à l'intérieur du contenu de page, sans repasser par le header.
5. Répéter le test sur une page non-home (ex: un article de blog) pour vérifier que ça fonctionne aussi avec `<Header />` (pas seulement `<HomeHeader />`).
6. Sans toucher au clavier, recharger la page et vérifier que rien de visible n'a changé par rapport à avant (le lien reste bien cascade tant qu'il n'a pas le focus).

git add src/layouts/Layout.astro
